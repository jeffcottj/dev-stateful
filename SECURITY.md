# Security Policy

## Reporting a Vulnerability

Please **do not** open a public GitHub issue for security vulnerabilities.

To report a security issue, email **security@example.com** with:

- A description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fixes (optional)

You will receive a response within 72 hours. If the issue is confirmed, we will work on a fix and coordinate a disclosure timeline with you.

## Supported Versions

Only the latest release on `main` is actively maintained for security fixes.

## Scope

This template is intended as a starting point for stateful web applications. Vulnerabilities in the template scaffold itself (dependencies, configuration patterns, CI setup) are in scope. Issues arising from how the template is extended or deployed are out of scope.
